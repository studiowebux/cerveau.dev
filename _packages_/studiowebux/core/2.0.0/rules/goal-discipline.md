# Goal Discipline Rules

Every goal must be a single, atomic, completable task. No exceptions.

## Goal Formula

**[Action] + [Specific Target] + [Clear Scope]**

WRONG: "Authentication system", "Improve performance", "Fix bugs in A, B, and C"
RIGHT: "Add session ID display to viewer header", "Create GET /api/brains endpoint returning JSON list"

## Before Creating a Goal

- Can this be broken down smaller? If yes, break it down.
- Can I complete this in one session? If no, too big.
- Is this ONE specific thing? If no, split it.
- Do I know exactly what "done" looks like? If no, clarify first.

## Working on a Goal

1. Set ONE task as in-progress in mdplanner
2. Work ONLY on that task
3. Complete it FULLY
4. Move to Pending Review + write progress note
5. ONLY THEN pick the next task

Never work on multiple tasks simultaneously. Never switch tasks without completing the current one. Never leave a task partially complete.

## Completing a Goal

A task is complete when:
- The specific work is 100% done
- All code works end-to-end
- No TODOs or follow-ups remain
- A `[progress]` note has been written to mdplanner
- The task is moved to "Pending Review" in mdplanner (owner moves to Done after verification)

## Red Flags

"While I'm here, let me also..." "This goal includes X, Y, and Z." "This touches multiple systems." STOP. Break it down into separate mdplanner tasks.
